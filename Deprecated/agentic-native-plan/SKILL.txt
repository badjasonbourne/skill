# Agentic Native Plan

## 核心哲学

> 给 Agent 交代任务的时候一定说清楚怎么验证，然后就不怎么需要管中间结果了。

开发计划的价值不在于描述「怎么实现」，而在于为最终目标定义一个**真实、可被 Agent 自主执行、可观测终态**的验收方案。实现路径和中间过程都交给执行的 Agent 自由发挥；你只关心最终结果：当验收通过时，目标就是真的达成了。

## 这个 Skill 产出什么

一份以**验收方案（Acceptance）**为核心的开发计划，而非实现步骤的罗列。一句话判断计划质量：把它丢给一个看不到中间过程的 Agent，它能不能独立判断最终目标达成了没有？能，则计划合格。

## 设计验收方案的四条原则

1. **真实优先（Real over Mock）**：验收必须打到真实系统——真实的浏览器、真实的大模型请求、真实的接口与数据库。禁止用 mock / stub / 假数据来「证明」功能可用。Mock 只能用于实现内部，不能作为验收手段。
2. **不止步于静态检查**：lint / typecheck / 编译通过 / 单元测试绿，只是**底线**，不是验收。它们证明「代码长得对」，不证明「行为是对的」。验收必须观测到运行时的真实行为。
3. **可观测的终态（Observable End State）**：验收要落在外部可见的事实上——一个网络请求的状态码与响应体、页面上出现的文本、数据库里新增的一行、模型返回的结构化字段。避免「看起来应该可以了」这类无法判定的表述。
4. **Agent 可自主执行**：验收步骤要能被 Agent 用手头工具（CDP / Chrome DevTools MCP、curl、运行脚本、查库等）独立跑完并读出结果，不依赖人工肉眼确认或人工点击。

## 验收方法清单

按目标类型选择真实的验收手段：

- **前端 / 交互行为** → 用 CDP / Chrome DevTools MCP 在真实浏览器里执行操作（点击、输入、跳转），然后：
  - 检查触发的**网络请求**：URL、方法、请求体、状态码、响应体是否符合预期；
  - 检查 DOM / 页面文本变化、console 有无报错；
  - 截图做视觉确认。
- **后端 API / 端到端** → 用 curl 或脚本发起**真实请求**，断言状态码与响应结构；必要时验证副作用（查真实数据库确认数据落库、查日志/队列确认事件产生）。
- **涉及大模型的功能** → 发起**真实的大模型请求**，验证返回是否满足约束（结构化字段、工具调用、流式事件顺序等）。不要用固定假响应替代。
- **数据 / 副作用** → 直接查真实数据存储核对结果，而非只看函数返回值。
- **CLI / 脚本 / 批处理** → 用真实输入运行，断言真实输出 / 退出码 / 产物文件。
- **静态检查** → 作为**附加底线**列出（编译、类型、lint、单测），但永远不能是唯一验收项。

## 工作流程

复制此清单跟踪进度：

```
- [ ] 1. 厘清目标的「外部可观测成功标准」
- [ ] 2. 写真实验收方案（方法 + 可判定的预期）
- [ ] 3. 自检：是否真实、可自主执行、可观测、非静态
- [ ] 4. 输出计划
```

**1. 厘清成功标准**：先问「这件事做成了，从外部看会是什么样？」用用户能感知或系统能观测的事实描述，而非内部实现。

**2. 写验收方案**：从上面的清单里为最终目标挑真实手段，写出**具体操作步骤**和**可判定的预期结果**。复杂目标可以列多条验收项，但它们共同验证的是同一个最终结果，而不是拆解中间任务。

**3. 自检**：逐条对照四条原则。任何一条出现 mock、只有静态检查、需要人工肉眼、或预期无法判定，都要重写。

**4. 输出计划**：用下方模板。

## 计划模板

```markdown
# {目标}

## 成功标准（外部可观测）
{做成后从外部看到的事实}

## 验收方案
- 方法: {真实手段，如「用 Chrome DevTools MCP 打开 /login，输入账号密码点击登录」}
- 预期: {可判定终态，如「触发 POST /api/login 返回 200，响应含 token；页面跳转到 /dashboard 且显示用户名；console 无 error」}
- （可补充更多验收项，共同验证最终结果）

## 底线检查
{编译 / 类型 / lint / 单测}
```

## 反模式

- ❌ 验收 = 「跑通单测 / 类型检查通过」 → 只是底线，没观测真实行为。
- ❌ 用 mock 的接口 / 假的模型响应来证明功能可用。
- ❌ 验收写成「确认功能正常工作」这类无法判定的话。
- ❌ 验收依赖人工肉眼检查或人工操作，Agent 没法自己跑完读结果。
- ❌ 计划里堆满实现步骤、却没有任何一条说清「怎么验证做对了」。
