# Linear Style

本指南将指导如何设计使得界面的UI和交互对齐 Linear 的设计语言。

## 启发式经验

- 避免使用全大写的单词（包括 label、tag、按钮文案、section 标题）。
- 避免使用太多卡片。一屏里 card 数量应该是"克制的"，能用分隔线、留白、分组标题表达层级时优先用它们。注意：区分页面分区的“大卡片”不在讨论范围内，比如左、右侧边栏和中间的主工作区等。我们更多指的是局部小卡片。
- 尽量避免使用阴影。
- 不使用状态圆点。圆点仅在确实表示"实时在线/运行中"等场景出现
- 优先使用较小的按钮来提升精致感
- 减少视觉上卡片套卡片的设计。
- 避免过多的边框引发视觉疲劳。
- 避免过多的辅助解释信息，而是让内容自解释。比如，避免表单中每个区域标题下的解释；避免卡片中不必要的辅助标签。

## 视觉准则

1. **按钮**: 按钮尺寸不宜过大，通常其高度约为内部文字字体大小的1.5倍。例如，文字使用14px时，按钮的上下内边距可设为4px左右。无需严格遵循两倍比例，找最接近且**偏小**的 token，不要四舍五入到偏大。
2. **卡片/Modal**: 卡片/modal分为Header、Body、Footer 三个部分，其中Header和Body是必选，Footer是可选的。Header右侧需含关闭按钮，使用remixicon close-line圆形图标按钮，圆心与卡片/modal圆角重合；左侧默认需要有标题。**三者间默认不添加分隔线，靠亲密性原则实现视觉层级**。
3. 合理统一局部内容轴: 无论容器层级如何，让局部容器中所有文字左缘对齐同一条垂直基准线，来构建视觉统一性。
4. **滚动条**: 不使用原生滚动条，请用更轻更细更短的滚动条，且不需要导轨。
