# Modern SaaS UX

面向初创团队的现代 SaaS 视觉风格指南。对齐 Linear、Cursor、Craft 的克制美学。

## 使用方式

在做任何 UI 实现或走查前：

1. 先按下面的「反 AI 味清单」逐项排查现有/计划中的设计
2. 再按「视觉准则」决定具体 token（spacing / 字号 / 颜色 / 边框）
3. 输出前做一次自检（见末尾 checklist）

## 启发式经验

- 避免使用全大写的单词（包括 label、tag、按钮文案、section 标题）。需要强调用字重 / 字号 / 颜色，而不是 uppercase
- 避免使用太多卡片。一屏里 card 数量应该是"克制的"，能用分隔线、留白、分组标题表达层级时优先用它们
- 尽量避免使用阴影。
- 不使用状态圆点。圆点仅在确实表示"实时在线/运行中"等场景出现
- 优先使用较小的按钮来提升精致感
- 不出现卡片套卡片。
- 避免过多的边框引发视觉疲劳。

## 视觉准则

1. **按钮**: 按钮尺寸不宜过大，通常其高度约为内部文字字体大小的两倍。例如，文字使用14px时，按钮的上下内边距可设为7px左右。无需严格遵循两倍比例，找最接近且**偏小**的 token，不要四舍五入到偏大。
2. **颜色与对比度**: 整体走"较低对比但和谐"的路线；但**关键文字必须保留明显对比度**。强调色（品牌色）只在 CTA、关键链接、当前选中态出现，单屏出现次数 ≤ 3。Hover 态用背景色变化而非边框出现/颜色加深。

## 自检 Checklist

提交前逐项确认：

- [ ] 全文检查没有任何全大写英文单词
- [ ] 一屏内卡片数量克制，没有卡片套卡片
- [ ] 没有阴影做层级（除非是 modal/popover overlay/dropdown等必要悬浮容器）
- [ ] 没有装饰性状态圆点
- [ ] 按钮 padding 走的是"偏小"的 token，不是默认大尺寸
- [ ] 强调色出现 ≤ 3 处
