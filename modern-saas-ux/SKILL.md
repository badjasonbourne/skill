# Modern SaaS UX

面向初创团队的现代 SaaS 视觉风格指南。对齐 Linear、Cursor、Craft 的克制美学，主动规避典型 AI 生成 UI 的廉价感。

## 使用方式

在做任何 UI 实现或走查前：

1. 先按下面的「反 AI 味清单」逐项排查现有/计划中的设计
2. 再按「视觉准则」决定具体 token（spacing / 字号 / 颜色 / 边框）
3. 输出前做一次自检（见末尾 checklist）

## 反 AI 味清单（硬性规避）

以下都是常见 AI 生成 UI 的廉价信号，默认全部不要：

- 不使用全大写英文单词（包括 label、tag、按钮文案、section 标题）。需要强调用字重 / 字号 / 颜色，而不是 uppercase
- 不堆砌卡片。一屏里 card 数量应该是"克制的"，能用分隔线、留白、分组标题表达层级时优先用它们
- 不使用阴影做层级。需要层级时用背景色差异（如 `bg-white` vs `bg-neutral-50`）或一根 1px 边框，二选一
- 不使用状态圆点（绿点 / 红点 / 黄点）作为常规标识。状态用文字 + 颜色表达，圆点仅在确实表示"实时在线/运行中"等场景出现
- 白色背景上禁止使用过淡的灰色文字。正文最低 `neutral-700`，次要信息最低 `neutral-500`，禁止使用 `neutral-300/400` 作为可读文本
- 不出现卡片套卡片。嵌套结构通过留白 + 内部分组（如 section heading + divider）表达
- 不滥用边框。按钮、卡片、Tab、Input 不要每个都加 border；优先用背景色 + hover 背景色代替

## 视觉准则

### 按钮

- 高度：上下 padding ≈ 文字字号的一半。常用映射：

  | 文字字号 | 推荐 padding (y) | Tailwind |
  |---------|-----------------|----------|
  | 12px    | 5–6px           | `py-1.5` |
  | 13–14px | 6–7px           | `py-1.5` 或 `py-2` |
  | 15–16px | 7–8px           | `py-2` |

- 找最接近且**偏小**的 token，不要四舍五入到偏大。
- 水平 padding 通常是垂直的 2.5–3 倍（如 `py-1.5 px-3`）。
- 主按钮：实色背景 + 无边框；次按钮：透明背景 + hover 浅灰背景（如 `hover:bg-neutral-100`），不加边框；幽灵按钮同理。
- 圆角统一用 `rounded-md`（6px）或 `rounded-lg`（8px），不混用。

### 边框

- 默认能不加就不加。需要分隔时优先级：留白 > 背景色差异 > 1px 边框。
- 同一屏内边框使用次数 ≤ 3 处为佳。
- 边框色用低对比度灰（如 `neutral-200`），但不要再叠加阴影。

### 颜色与对比度

- 整体走"低对比但和谐"的路线，让画面融为一体；但**关键文字必须保留明显对比度**：
  - 标题 / 主文案：`neutral-900` 或 `neutral-800`
  - 正文：`neutral-700`
  - 次要 / 辅助：`neutral-500`（白底下的下限）
  - 禁用：`neutral-400`（仅用于真正 disabled 状态）
- 强调色（品牌色）只在 CTA、关键链接、当前选中态出现，单屏出现次数 ≤ 3。
- Hover 态用背景色变化（`hover:bg-neutral-100` / `hover:bg-neutral-50`）而非边框出现/颜色加深。

### 间距与层级

- 同组元素间距小（如 `gap-2`），跨组间距大（如 `gap-6` / `gap-8`），用 spacing 而不是分割线表达分组。
- section 标题用字重 + 字号区分（如 `text-sm font-medium text-neutral-900`），不要加背景条 / 大边框。

### Tab 与导航

- Tab 用底部 2px 实色横线 + 文字颜色变化，不加外框、不加背景色块。
- 侧边栏菜单项用 hover 背景 + 选中态背景，不加左侧 indicator 条（除非项目已有规范）。

### Input / Select

- 默认单边或无边框 + 浅灰背景；focus 时换背景色或加细 ring，不要同时加边框 + 阴影 + ring。

## 自检 Checklist

提交前逐项确认：

- [ ] 全文检查没有任何全大写英文单词
- [ ] 一屏内卡片数量克制，没有卡片套卡片
- [ ] 没有阴影做层级（除非是 modal/popover overlay）
- [ ] 没有装饰性状态圆点
- [ ] 白底上没有 `neutral-300/400` 充当正文
- [ ] 按钮 padding 走的是"偏小"的 token，不是默认大尺寸
- [ ] 单屏边框使用 ≤ 3 处
- [ ] 强调色出现 ≤ 3 处
- [ ] 关键文字对比度足够（标题 `neutral-900`，正文 `neutral-700`）

## 何时可以破例

- Modal / Popover / Dropdown 的 overlay 阴影是合理的（用于浮层而非层级）
- 数据密集型表格里，1px 行分隔是合理的
- 真正的实时状态（在线 / 运行中 / 录制中）可以用圆点

破例时在 PR 描述或注释中明确说明原因。
