# 导航

## 导航控件与层级语义

> 导航（去别处 / 回上级）和操作（提交 / 完成）要用不同的控件；面包屑层级必须映射真实信息结构，可点击的只能是真实父级。

### 为什么

按钮——尤其实心/描边按钮——是「操作」的视觉语言，用户预期它会「做一件事」。把「返回」做成按钮，会和页面真正的主操作抢权重、混淆性质。导航应交给更轻的控件：面包屑、带 icon 的文本链接。面包屑还承担「我在哪、能回到哪」，所以它的层级必须和真实结构一致，否则误导。

### 反例 → 改进：返回用什么控件

反例 —— 返回上级和「完成」并排成两个按钮，导航和操作混在一起：

```text
[ 返回选题 ]   [ 完成练习 ]      ← 左边是导航却长得像操作
```

改进 —— 返回交给面包屑（或带 icon 文本链接），按钮只留给操作：

```text
练习 / 题目摘要                   ← 面包屑承载返回，「练习」可点
                  [ 完成练习 ]    ← 页面唯一操作按钮
```

### 反例 → 改进：面包屑层级语义

反例 —— 面包屑塞进一个不存在于真实结构里的中间段：

```text
练习 / 选择一道题 / 题目摘要       ← "选择一道题"不是真实层级，是动作
```

改进 —— 每一段都是真实父级；可点击的是用户能回到的那一层：

```text
练习 / 题目摘要                   ← "练习"是真实父级且可点，叶子为当前页
```

### 判断法

- 一个控件是「去别处/回上级」吗？是 → 用面包屑或文本链接，不要用实心/描边按钮。
- 面包屑每一段都对应一个真实存在、可回到的层级吗？把「动作名」当成层级 → 删掉它，让真正的父级可点击。
